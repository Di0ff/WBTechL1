Вопрос 1
Какой самый эффективный способ конкатенации строк?

Из тех методов, что я знаю, это builder, +, Join и Sprintf.

1. Оператор +
Самый эффективный способ для небольшого количества строк — оператор +.
Однако, если строк много, то памяти выделяется значительно больше,
так как создаются временные объекты на каждом шаге конкатенации.
Это увеличивает использование памяти и замедляет выполнение.

2. strings.Join
Если строки уже находятся в слайсе,
то Join будет самым удобным и эффективным способом,
так как он выполняет конкатенацию за одно выделение памяти.
Этот метод подходит для случаев, когда у вас уже есть срез строк.

3. strings.Builder
Builder — лучший вариант для большого количества строк,
так как он использует внутренний буфер для уменьшения количества выделений памяти.
Это делает его более эффективным по сравнению с оператором + для длинных строк или множества строк.

4. fmt.Sprintf
Sprintf хуже всех по эффективности,
так как он не только соединяет строки, но и форматирует их.
Этот метод удобен для форматирования,
но для простой конкатенации строк он менее эффективен из-за дополнительных затрат на обработку форматирования.

Вывод:
1. Для небольшого количества строк: оператор + может быть достаточным.
2. Для большого количества строк: предпочтительно использовать strings.Builder или strings.Join.
3. Для форматирования строк: fmt.Sprintf подходит для сложного форматирования, но не для простой конкатенации.
Таким образом,
выбор метода конкатенации строк зависит от конкретного случая и объема данных.
